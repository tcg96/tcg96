TrollRAT

TrollRAT is an open-source remote administration tool developed by Leurak. It's designed to make the computer appear messed up while a scammer pretending to be a technician is connected to it remotely. For more info, check out the GitHub repository. You can also look up "trollrat" on YouTube for various demonstration videos.

Download TrollRAT

Please do not use it for malicious purposes.

How to set up TrollRAT:

  1. Extract the files from TrollRAT.zip (they should be TrollRAT.exe and a Plugins folder) wherever you want, but make sure they're in the same folder.
  2. Run the TrollRAT executable. If it asks for administrative permissions, say yes.
  3. Nothing should show up, it'll run in the background. To make sure it's running, check the "processes" section of Task Manager.
  4. On another computer, or on your host machine if you ran TrollRAT on a virtual machine, open a web browser and visit the local IP address of the computer TrollRAT is running on, on port :1337.
    • E.G.: if TrollRAT is running on a machine with the local IP address 192.168.0.2, type 192.168.0.2:1337 in your web browser.
  5. You're ready to have fun.

How to quit TrollRAT (stop it from running in the background):

End its process from Task Manager, or restart your computer.

If no plugins/payloads show up:

  1. Quit TrollRAT from Task Manager.
  2. Go to the folder where the TrollRAT files are.
  3. Right click on TrollRAT.exe, click Properties, and then Unblock.
  4. Repeat step 3 for the three files in the Plugins/TrollRATPayloads folder. If you don't see any files there, you downloaded a corrupt or incomplete copy of the program.

If you don't know your machine's local IP address:

  1. Right click on the network icon on your taskbar's system tray.
  2. Select "Open Network and Sharing Center".
  3. On the right-hand side, where it says "Connections:" click on your connection (it could be called Local Area Connection, Ethernet, Wi-Fi, etc.)
  4. Click "Details".
  5. Note the "IPv4 Address" (or "IPv6 Address" if you use IPv6). That's your local IP address.

If that doesn't work or you have Windows XP, open the Command Prompt, type "ipconfig" and press Enter. You'll see your local IPv4 or IPv6 address somewhere in there.

If you're sure you've got the right IP address of your virtual machine, but it doesn't work, change your VM's networking settings from "NAT" to "Bridged". It's not always required, but it will fix this problem if you have it.
Warning: Switching the VM's networking to bridged will let the VM see other devices in your local network, as if the VM was connected to your router directly. If you have any servers or anything in your local network (e.g. NAS), make sure they're password protected or offline, and if the VM asks for a location of the network, choose "Public".
